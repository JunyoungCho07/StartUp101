```
셀프 계산대를 선호하지만, 바코드 오류·성인 인증·쿠폰 처리 때문에

[3. 고객의 핵심 고통 (Constraints) — 인터뷰·설문 raw quote 기반]
P1. 바코드가 안 찍혀 결국 직원을 부르게 됨 (전체 셀프 사용자 44%가 경험)
P2. 주류·성인 인증 시 직원 호출 → 비대면 가치 훼손 (15%)
P3. 쿠폰·상품권 적용 시 헷갈림 → 직원 호출 (15%)
※ 셀프 선호자 21명 중 17명(81%)이 불편 경험 보고

[4. 핵심 가치 제안 (UVP)]

- 쿠폰·상품권을 자동 매칭

  Card 3: "바코드가 누락된 채 결제하고 매장으로 되돌아간 적이 있어요. 매번 불안함."
```

## 추가 v0.dev 후속 명령 (1차 결과 받은 후 사용)

### 명령 1 — 이미지 교체 (v0는 placeholder 생성하므로)
```
Hero 우측 모바일 앱 mockup 이미지를 다음 URL로 교체해줘:
https://source.unsplash.com/random/600x800/?smartphone,shopping,korean
```

### 명령 2 — sticky header 추가
```
페이지 상단에 sticky header를 추가해줘.
좌측: NoCall 로고 텍스트, 우측: "베타 사전등록" CTA 버튼.
스크롤 시 흰색 배경 + 그림자.
```

### 명령 3 — 전환 추적 코드 자리 (선택)
```
Final CTA 폼 제출 성공 시 "감사합니다 — 곧 초대장을 보내드립니다" 모달을 띄우고,
window.dataLayer.push({event: 'beta_signup'}) 코드를 삽입해줘.
(Google Analytics 또는 Plausible 연결용 hook)
```

## A/B 테스트 후속 계획

베타 등록 8% 미만일 경우 다음 변형 테스트:

- **Variant B**: 헤드라인 변경 → "셀프 계산대 90초, 매번 직원 호출에 막히시나요?"
- **Variant C**: CTA 변경 → "베타 신청" → "30초 무료 체험 신청"
- **Variant D**: Pain Mirror 위치 → Hero 바로 아래 → Solution 다음으로

각 variant는 1주 단독 운영, 통계적 유의성(p < 0.05) 확보될 때까지 최소 n=200/variant.
