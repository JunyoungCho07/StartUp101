## 1. 인사이트 1 — "셀프 계산대 선호 + 불편 공존" 역설

### 핵심 수치
- 셀프 선호자 **21명 중 17건(81%)** 이 불편 경험을 보고함
- 이들은 셀프를 "좋아서"가 아니라 **"덜 나쁜 선택지"** 로 사용 중

### 셀프 선호 이유 원문 (전체 21건)
```
[1]  비교적 짧은대기
[2]  대기시간없이 바로 계산할수있어 편리함
[3]  셀프 계산대가 더 많음
[4]  간편하거나 사람이없움
[5]  극 I라서 사람 대면하기가 두렵습니다
[6]  편해서
[7]  혼자서 하는 것이 편하고. 소비 내역을 확인하기 쉽다
[8]  혼자하면 되어서 편해서
[9]  키오스크 사용에 어려움이 없고, 대기줄이 거의 없기 때문에
[10] 뭔가 내가 이 물건들을 직접 샀다는 느낌이 더 들어서 심적으로 편하다.
[11] 대기 시간이 더 적기 때문
[12] 비대면이 편해요
[13] 사람이 싫음. 사람이 계산을 해줄 때는 뭔가 약간의 부담이 있는데 셀프 계산대는 그런 부담을 안느껴도 돼서 좋음.
[14] 직접 계산하는게 더 신뢰성 높고 덜 부담스러워요
[15] 사람과 상호작용하기 싫어서
[16] 직원이 불친절하면 기분이 나빠서, 그리고 내가 불친절하게 하면 직원도 기분이 나쁠까봐 굳이 사람한테 계산을 부탁하고 싶지 않음.
[17] 포인트 적립과 주차 여부와 같은 질문을 받지 않고 원하는 결제 방식으로 빠르게 결제할 수 있기 때문입니다.
[18] 점원과 상호작용이 없는 것이 편함
[19] 신속하게 구매할 수 있기 때문
[20] 편함
[21] 내가 하는게 더 편하다
```

### 해석
셀프 선호 이유는 크게 두 축으로 분류된다:
- **속도/편의 축** (1,2,3,6,9,11,19,20,21): 대기가 짧고 빠르기 때문
- **비대면/심리 축** (4,5,7,8,10,12,13,14,15,16,17,18): 사람 대면이 불편하거나 부담스럽기 때문

**진짜 동기는 기술 친화성이 아닌 사회적 마찰 회피(Social Friction Avoidance)다.**

## 2. 인사이트 2 — 셀프 계산대 Pain Point 카테고리화

### 수치 요약

| 불편 카테고리 | 건수 | 비율 |
|---|---|---|
| 바코드/인식 오류 | 12 | 44% |
| 직원 호출 필요 | 9 | 33% |
| 상품권/쿠폰 처리 어려움 | 4 | 15% |
| 주류/성인인증 | 4 | 15% |
| 취소/환불 불편 | 3 | 11% |
| 정확성 불안(누락 불안) | 3 | 11% |
| 터치 딜레이 | 1 | 4% |
| 카드 오류 | 1 | 4% |

> 총 불편 응답 27건 기준. 복수 카테고리 해당 응답 포함.

### 셀프 선호자의 불편 경험 원문 (21명 중 17건)
```
- 기계에 따라 상품권 할인 등의 반영이 어려울 때가 있음
- 주류구매시 직원호출후 미성년자여부 확인후 계산가능함 불편
- 인식오류
- 클릭 딜레이가 좀 거슬려요(터치가 즉각적으로 반영이 안되고 딜레이가 있어서 조작감이 불편하고 작업 시간이 오래걸림)
- 직원의 도움이 필요한 몇몇 상품이 있다
- 상품권 이용을 하고 싶을 때 종류가 너무 많아서 도대체 뭘 눌러야 하나 헷갈림
- 종종 바코드 인식이 되지 않는다
- 바코드 인식 오류
- 바코드가 한번에 안찍힐 때 조금 짜증
- 쿠폰 적용이 안될 때가 있어요
- 미성년자 구입 제한 상품의 경우 결국 직원을 불러야 해서 번거로움
- 다이소 같은 곳에서는 직원 확인이 있어야 잘못 찍은 물품을 취소할 수 있는데, 불편함.
- 바코드 인식이 누락되어 매장으로 되돌아가 추가적으로 결제한 적이 있습니다. 이러한 실수를 현장에서 알지 못한다면 책임을 온전히 져야 한다는 점이 부담스러운 것 같습니다.
- 잘못 계산했을 때 즉시 환불이 어려움
- 바코드 인식 오류, 주류 구매시 직원 호출, 카드 인식 오류
- 상품에 바코드가 어딨는지 찾느라 시간이 지체됨
- 채소나 상품권, 쿠폰 사용할때 직원 도움이 필요하다.
```

### 핵심 구조적 모순
> "셀프 계산대인데 결국 직원이 필요하다"
> 바코드 오류(44%) → 직원 호출 / 주류·성인인증(15%) → 직원 호출 / 취소(11%) → 직원 호출
> 세 카테고리 모두 **직원 호출**로 귀결 = 셀프의 본질적 가치(비대면·자율성)를 훼손

## 4. 인사이트 4 — 20대 셀프 선호의 심리적 핵심: 비대면

셀프 선호 이유 중 **비대면/사회적 마찰 회피** 범주 직접 인용:

```
"극 I라서 사람 대면하기가 두렵습니다"
"사람이 싫음. 사람이 계산을 해줄 때는 뭔가 약간의 부담이 있는데 셀프 계산대는 그런 부담을 안느껴도 됨."
"직원이 불친절하면 기분이 나빠서, 그리고 내가 불친절하게 하면 직원도 기분이 나쁠까봐 굳이 사람한테 계산을 부탁하고 싶지 않음."
"포인트 적립과 주차 여부와 같은 질문을 받지 않고 원하는 결제 방식으로 빠르게 결제할 수 있기 때문."
"사람과 상호작용하기 싫어서"
"점원과 상호작용이 없는 것이 편함"
```

> 이는 단순 UX 불편이 아닌 **사회적 마찰(social friction) 회피 욕구**. 솔루션 방향은 "속도"보다 "직원 개입 최소화"가 더 강력한 가치 제안이 됨.

## 5. ICGC 가설 vs 실제 데이터 정합성 검증

### [I] Identity — 부분 지지, 정의 수정 필요

| 가설 | 데이터 |
|---|---|
| "디지털 기술 친화, 스마트 쇼핑객" | 셀프 사용 동기는 기술 친화가 아닌 **사회적 마찰 회피** |

### [C] Context — 가장 심각한 미스매치

| 가설 | 데이터 |
|---|---|
| 오후 6~8시, 마트/슈퍼마켓, 카트 쇼핑 | 저녁 방문 29명 중 편의점 압도적 다수 |
| 피크타임 대형마트 Pain | 대형마트+저녁+Pain 동시 충족: **4명(8.7%)** |

저녁(오후7~10시) 방문자 29명 중 매장 분포:
- 소형 편의점: 다수
- 대기 Pain 경험: 3명만 해당

### [C] Constraints — 수치 직접 충돌

| 가설 원문 | 실제 데이터 |
|---|---|
| "최대 30분씩 대기" | 최장 응답: **5~10분** (6명, 13%) |
| "극심한 좌절감, 구매 포기" | 구매 포기 언급: **0건** |
| "피크타임 주 방문" | 대형마트+저녁+Pain: **4명(8.7%)** |

> 가설의 Constraint 강도가 현실보다 3~6배 과장.

### [G] Goal — 근거 빈약

| 가설 원문 | 데이터 |
|---|---|
| "Just Walk Out / Checkout-free" | 직접 언급: 0건 |
| "카트 실시간 가격 확인" | 언급: 0건 |
| 실제 Goal | "빠른 결제" + "직원 안 만나도 되는 결제" |

## 7. 발표 전 선택 경로

### 경로 A — Pivot (가설을 데이터에 맞게 수정)

데이터가 강하게 지지하는 재설정 방향:

- **Identity**: 셀프 계산대를 선호하지만 반복적 오류로 결국 직원을 호출해야 하는 쇼핑객
- **Context**: 셀프 계산대 사용 중 바코드 오류 / 주류 성인인증 / 쿠폰·상품권 처리 시
- **Goal**: 직원 개입 없이 셀프 계산을 완결하는 것
- **Constraints**: 현재 셀프 계산대 UX의 구조적 한계 (직원 없이 해결 불가한 예외 케이스들)

지지 근거: 셀프 선호자 21명 중 81% 불편 경험 / 불편의 77%(바코드+직원호출)가 동일 구조

### 경로 B — Persevere + 데이터 부족 인정

"이번 설문은 타깃 세그먼트 커버리지 부족 (n=4). 50대 + 대형마트 이용자 대상 추가 현장 인터뷰 3~5건 진행 계획" 선언.
단, 현재 데이터로 Persevere 근거 불충분 → 발표 평가상 약점 노출.
