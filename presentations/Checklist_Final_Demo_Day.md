## PART 1 — 고객 및 문제 정의 [권장 2~3장]
> 소스: `Final_Presentation_Checklist.txt` §1 · `Week14_Storytelling_IR.txt` Step1

### 필수 슬라이드 항목
- [ ] **ICGC 페르소나 업데이트** — 중간발표 이후 추가 인터뷰 결과로 갱신된 페르소나 제시
  - 기존: "주 3회 이상, 50대, 대형·창고형 마트, 피크타임 6~8시"
  - 추가 필요: 실명 or 가명 + 직접 인용 한 줄 ("○○님이 이렇게 말했습니다")
- [ ] **문제의 본질** — 셀프계산대 역설 (직원호출 81%, 바코드오류 44%) 그래프/수치로 강조
- [ ] **Why Now?** — 왜 지금 이 문제를 풀어야 하는가?
  - 힌트: 셀프계산대 보급률 증가 추세, 인건비 상승 구조, 마트의 셀프 전환 압력
- [ ] **고객·문제 검증 데이터** — n=46 설문 + 현장 인터뷰 횟수 수치화
  - "인터뷰 총 N건, 설문 46명, 7/7 우회 행동 100%" 대형 폰트

### 🚨 감점 방지 (공식 체크리스트에서 발췌)
- [ ] 타깃이 "마트 이용 소비자 전체"처럼 광범위하지 않은가? → 50대 단독 세그먼트 명시
- [ ] 뉴스 기사 캡처가 아닌 **팀이 직접 수행한 인터뷰 원문 인용구**가 1개 이상 있는가?

### 🔴 중간발표 미충족 → 반드시 보완
- [ ] **현장 인터뷰 직접 인용 1건 이상** — 50대 인터뷰이의 실제 발화 원문 ("그 날 진짜 짜증났던 게...")
- [ ] **Pain 심각도 데이터** — "불편했다" 수준이 아닌 행동 변화 증거 (구매 포기, 채널 이탈 횟수)

## PART 2 — 솔루션 및 기술 [권장 2~3장]
> 소스: `Final_Presentation_Checklist.txt` §2 · `Week14_Storytelling_IR.txt` Step2

> 🟡 **중간발표와 가장 큰 차이점** — 중간발표는 "솔루션 없음"이었지만 기말은 **솔루션 필수**

### 필수 슬라이드 항목
- [ ] **핵심 솔루션 정의** — NoCall의 핵심 가치 제안 1줄
  - 현재 후보: "셀프계산대에서 직원 호출 없이 모든 계산을 완결하는 앱"
  - 공식 가이드: "고객의 진짜 문제를 정확히 관통(Problem-Solution Fit)해야 함"
- [ ] **솔루션 검증 (2차 인터뷰)** — MVP/와이어프레임을 고객에게 시연하고 받은 피드백
  - `product-design/System_Prompt_NoCall.md` 의 v0.dev 랜딩페이지가 MVP로 활용 가능
  - 필요: 랜딩페이지를 실제 타깃에게 보여주고 반응 수집
- [ ] **제품/서비스 핵심 화면** — 최소 1~2개 UI 스크린 또는 서비스 플로우 다이어그램
- [ ] **피드백 반영 히스토리** — "초기 가설 → 피드백 → 어떻게 진화했는가"
  - 예: Just Walk Out → (데이터 기각) → 직원호출 최소화 앱 → 현재 NoCall

### 🚨 감점 방지
- [ ] 솔루션이 Part 1에서 정의한 셀프계산대 역설 문제를 직접 해결하는가?
- [ ] 아이디어 자랑에 그치지 않고 "고객 피드백을 통한 린(Lean) 발전 과정"이 보이는가?

### 🔴 현재 상태 → 필요 액션
- [ ] **Commitment 확보** — v0 랜딩페이지 베타 등록 또는 인터뷰 후 "다음 주 시연 약속" 캘린더 확정
- [ ] 베타 사전등록 수 집계 (목표: 전환율 8%, `product-design/System_Prompt_NoCall.md` 기준)

## PART 3 — 시장성과 비즈니스 모델 [권장 2~3장]
> 소스: `Final_Presentation_Checklist.txt` §3 · `Week13_Business_Model.txt` · `Week14_Storytelling_IR.txt` Step3

### 필수 슬라이드 항목

#### 3-1. 비즈니스 모델 (수익 구조)
- [ ] **수익 모델 패턴 선택** — 6가지 중 하나 이상 명시
  | 패턴 | NoCall 적용 가능성 |
  |------|------------------|
  | 구독 (Subscription) | 마트 운영사 월정액 SaaS |
  | 플랫폼 (Platform) | 마트-소비자 중개 |
  | SaaS | 마트 운영사 대상 셀프계산 솔루션 |
  | 프리미엄 (Freemium) | 소비자 무료, 고급기능 유료 |
- [ ] **User vs Customer 명확 구분**
  - User: 마트 소비자 (셀프계산대 이용자)
  - Customer (돈 내는 사람): 마트 운영사? 소비자? → 명확히 정해야 함
- [ ] **구체적 Pricing 시뮬레이션** — "수수료를 받겠다" 수준이 아닌 구체적 금액 예시
  - 예: "마트 1개 점포당 월 ○○만원" 또는 "거래 건당 ○○원"

#### 3-2. 시장 규모 (TAM-SAM-SOM)
- [ ] **Bottom-up 방식으로 계산** — Top-down(시장 1% 점유) 방식 사용 금지
  - 1단계: 단위 가격 (ACV) 설정
  - 2단계: SOM 산출 — 즉시 도달 가능한 유효 고객 수
  - 3단계: SAM 확장 — 국내 마트 업계 전체
  - 4단계: TAM 도출 — 글로벌 셀프계산대 시장
  - 참고 수치: 국내 대형마트 점포 수, 연간 셀프계산대 거래 건수
- [ ] TAM-SAM-SOM 3단계 시각화 (파이·깔때기 다이어그램)

#### 3-3. 경쟁 환경 및 우위
- [ ] **기존 대안(우회 수단) 정리** — 오픈런, 온라인 전환, 줄 바꾸기
- [ ] **경쟁사 비교** — 2×2 매트릭스 또는 비교표
  - 비교 축 예시: "직원 개입 여부" × "셀프 완결성"
- [ ] NoCall의 독점적 차별화 1줄 명시

### 🚨 감점 방지
- [ ] "수수료를 받겠다" 수준의 모호한 BM이 아닌가?
- [ ] TAM-SAM-SOM을 단순 통계 보고서 수치로 나열하지 않았는가? (Bottom-up 필수)
- [ ] 경쟁사를 단순 비방하지 않고 객관적으로 한계를 인정하면서 우위를 증명하는가?

### 🔴 NoCall 특이사항
- [ ] 비즈니스 모델 캔버스(BMC) 9개 블록 내부 초안이라도 작성 (심재후 교수 언급)
- [ ] LTV:CAC 비율 — SaaS 패턴 선택 시 3:1 이상 목표 제시
