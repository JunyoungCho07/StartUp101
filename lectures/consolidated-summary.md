## 1. 과목 개요 및 철학

**핵심 철학: "만들기 전에 판다 (Sell Before You Build)"**

아이디어 발상 → 제품 구현 → 고객 개발까지 창업 전 과정을 15주 안에 압축 경험하는 프로젝트 기반 창업 입문 과목. 단순 이론이 아닌, 시장에서 통하는 '팔리는 문제'를 먼저 정의하고 검증한다.

**4대 교육 축:**

- **실전적 철학 (Sell First):** 솔루션보다 문제를 먼저 정의·검증
- **창업 전 과정 압축:** 아이디어 발상 → 노코드 제품 구현 → 고객 개발 → 15주 완주
- **AI 가속화 (AI-Driven):** 코딩·디자인 역량 없어도 생성형 AI로 실제 작동하는 서비스 구현·배포
- **기후테크 특화 (Climate Tech):** 탄소 감축 등 거시적 환경 변화 속 비즈니스 기회 포착

## 2. 학습 목표

1. **AI 기반 창업 역량:** 생성형 AI로 BM 설계, 페르소나 분석, 프로토타입 제작, 마케팅 자동화
2. **시장 검증 체화:** 고객 인터뷰·실험으로 '증거(Evidence)'를 확보하는 린 스타트업(Lean Startup) 방식
3. **창업가적 문제 해결:** 기후 위기·사회 문제 속에서 기회를 포착해 비즈니스 솔루션으로 구체화
4. **MVP 구현 및 배포:** 노코드·AI 코딩 툴로 실제 작동하는 제품을 시장에 배포
5. **데이터 기반 의사결정:** 고객 반응 데이터 분석 → 피벗/전진 결정 → 투자자 대상 피칭

## 5. 1주차: 오리엔테이션 핵심 내용

### 창업가 DNA 3유형 (Hacker-Hipster-Hustler)

- **Hacker (기술):** 어떻게 만들 것인가?
- **Hipster (디자인):** 어떻게 매력적으로 전달할 것인가?
- **Hustler (비즈니스):** 어떻게 시장에 팔 것인가?

기능적 기술 연마보다 **'인사이트(Insight)'** 를 발굴하는 것이 핵심.

### 이펙추에이션 (Effectuation)

미래를 예측하는 대신, **현재 내가 가진 자원(나, 지식, 네트워크)으로 통제 가능한 결과를 만들어내는** 사고방식.

### 팀 빌딩 방식

1. **[실습 1] AI 코치와 함께하는 '나' 탐색**
   - AI 창업 코치와 1:1 심층 대화
   - 입력: 전공, 관심사, 강점, 이펙추에이션 성향
   - 출력: AI가 분석한 객관적 DNA 리포트 (13가지 유형 분류)
   - 목표: 내 강점 파악 + 팀 빌딩 시 '보완이 필요한 퍼즐 조각' 정의

2. **[실습 2] 30초 Reverse Pitching**
   - Identity (어떤 유형의 창업가인가?)
   - Asset (팀에 즉시 기여할 수 있는 역량)
   - Wanted (내 빈자리를 채워줄 사람은 누구인가?)
   - **핵심:** 아이템을 팔지 말고, 나 자신을 세일즈하라

3. **교수진 주도 최적화 팀 빌딩 (AI-Matching)**
   - 비슷한 사람이 모이는 것 방지
   - 상호 보완적 역량(3유형)이 완벽히 섞이도록 교수진이 직접 편성
   - 3주차에 최종 팀 명단 공지

## 6. 5주차: AI 활용 고객 발굴 및 인터뷰 실습

### 핵심 원리: 기술의 함정 — 솔루션 우선 vs 문제 우선

```
솔루션의 질을 높인 뒤 문제를 찾는 방법은 없다.
스타트업이 가장 주력해야 할 일은 "해결할 가치가 있는 문제"를 찾는 것.
```

- **나쁜 사례 — 구글 글래스:** 기술적 참신함에 집중, 고객의 실제 문제 부재
- **좋은 사례 — 애플 워치:** 초기 모호한 가치 제안 → 고객 맥락(건강/피트니스) 중심 피벗

### 확증 편향 (Confirmation Bias)의 위험

사람은 자신이 보고 싶은 것만 본다. 창업가는 자신의 아이디어가 옳다는 정보에만 주의를 기울이는 뇌의 특성을 가진다. 린 캔버스의 Plan A는 어디까지나 **'가설'** — 현장의 증거(Evidence)로 검증해야 한다.

### 페르소나 설계 원칙

뭉뚱그린 시장(예: '20-30대 남성')을 버리고, **압도적 지지를 받는 구체적 페르소나** 에 집중.

**[Case Study] 혼다 슈퍼 커브:** '모든 사람'이 아닌 '소바 가게 배달원(한 손에 배달통을 든 상황)'이라는 구체적 페르소나에 최적화 → 폭발적 반응.

**ICDO 페르소나 프레임워크:**

| 요소 | 질문 |
|------|------|
| Identity (정체성) | 문제를 겪는 구체적인 주체는 누구인가? |
| Context (상황/맥락) | 어떤 상황에서 그 문제가 Pain Point로 발화되는가? |
| Goals (목표) | 고객은 이 문제를 해결함으로써 어떤 상태에 도달하고 싶은가? |
| Constraints (제약 조건) | 왜 기존의 대안책들은 작동하지 않는가? 왜 아직도 해결하지 못했는가? |

### 고객 여정 지도 (Customer Journey Map)

```
여행 계획 → 준비·예약 → 이동·도착 → 현지 활동 → 여행 후 공유
    ↓              ↓           ↓            ↓             ↓
  사고(Think)   감정(Feel)   행동(Do)    고통(Pain)   [불편함이 분노에 도달하는 상황]
```

고객의 정량적 행동뿐만 아니라, **이면에 숨겨진 비합리적인 감정선(불편, 불쾌, 불안)** 을 가시화.

### [실습 1] NotebookLM 활용 페르소나 도출 (40분)

1. **[Step 1]** 타깃 시장·잠재 고객 관련 자료 수집 (기사, 리서치, 인터뷰 녹취, 유튜브 등 최소 8개 소스)
2. **[Step 2]** 수집한 텍스트·영상 소스를 NotebookLM에 업로드
3. **[Step 3]** AI로 초기 페르소나 후보 도출
4. **[Step 4]** 현실에 존재할 법한 입체적인 '가상 유저 프로필' 문서화

### 인터뷰 질문 설계 — 나쁜 질문 vs 좋은 질문

| 나쁜 질문 ✗ | 좋은 질문 ✓ |
|------------|------------|
| "이 제품 어때요?" | "지난번에 그 문제가 발생했을 때 어떻게 대처하셨나요?" |
| "이 기능이 있으면 편할 것 같지 않나요?" | "현재 업무에서 가장 비효율적인 부분은 무엇입니까?" |
| "이게 큰 문제인가요?" | "마지막으로 이 문제를 겪었을 때의 상황을 단계별로 설명해 주시겠어요?" |
| 미래 의향 질문 (거짓말 구역) | 과거 행동 요구 (사실 발화) |

**[마법 램프 질문]:** "만약 무엇이든 들어주는 마법 램프가 있다면, 현재 겪는 불편함을 해결하기 위해 어떤 완벽한 솔루션을 요청하겠습니까?"

### [실습 2] 가상 유저(Simulated Personas) 심층 인터뷰 롤플레잉 (50분)

도출된 페르소나 데이터를 AI(Claude/GPT/ChatGPT)에 주입 → '가상 잠재 고객' 생성 → 팀원들이 돌아가며 문제 인터뷰 진행.

> **팁:** AI 답변이 너무 긍정적이면 "더 까다롭고 방어적인 태도로, 실제 예산이 부족한 사람처럼 다시 답변해 줘"로 객관성 강화.

### KJ법 (친화도법) 활용 진짜 원인 도출

1. **[Step 1]** 인터뷰 내용 카드화 (단순 정리 X, 표면적 Lip Service 이면의 '내재된 진심'과 '진짜 원인' 도출)
2. **[Step 2]** Bottom-up 클러스터링 — 그룹을 먼저 정하지 말고, 카드 간 상호 관련성을 찾아 논리적으로 묶기
3. **[Step 3]** 각 그룹의 핵심을 관통하는 '문제의 진짜 원인'을 한 문장으로 정의

### Javelin Board (가설 검증 추적 도구)

| 고객/문제 가설 (Hypothesis) | 실험 및 기준 (Experiment) | 결과 및 학습 (Result & Learning) |
|--------------------------|--------------------------|--------------------------------|
| 우리의 가설 기술 | 검증 방법 및 성공 기준 명시 | 검증/반증 여부 + 다음 액션 |

## 7. Customer Discovery 101 (NYU, Frank Rimalovski)

### 핵심 용어 정의

- **MVP ≠ Product** (MVP는 최소 기능 제품, 완성품이 아님)
- **Pivot ≠ Restart** (피벗은 방향 전환, 처음부터 다시가 아님)
- **Customer Discovery ≠ Selling** (고객 발굴은 파는 게 아님)

### Customer Discovery의 목표

1. 문제/솔루션 적합성 (In)validate
2. 비즈니스 모델 구성 요소 가설 (In)validate
3. MVP 식별 및 테스트 시작
4. **목적:** 통계적 유의성이 아닌, **패턴 인식을 통한 인사이트 도출**

### 인터뷰 대상 찾는 법

- **최소 1단계 이상의 분리** (지인 직접 인터뷰 금지)
- 소개(Referral) 적극 활용
- 창의적 모집 방법 활용 (Fish where the fish are)
- 짧게 끝내겠다고 약속
- 학생/연구자 카드 활용

### 효과적인 인터뷰 원칙

- **팔지 마라. 데모 하지 마라. 발표하지 마라.**
- 사견이 아닌 **스토리**를 끌어내라
- **개방형 질문**을 던져라
- **왜? → 왜? → 왜?** 반복 (드릴다운)
- 스크립트가 아닌 **가이드** 작성
- **확증 편향**을 경계하라
- 반드시 **1:1 대면**으로 진행
- 침묵을 두려워하지 마라 (STFU)
- 노트 테이커를 따로 지정
- 인터뷰 직후 핵심 인사이트 바로 기록

## 8. 중간발표 (Proof Day) 가이드

### 발표 목적

완벽한 제품을 보여주는 자리가 아니다. **"고객이 진정으로 고통받는 문제가 존재하는지"를 증명**하는 자리.

### 발표 4단계 구조 (The Proof Day Journey)

#### Step 1. 가설 — 타깃 고객의 ICDO 설계도

**뭉뚱그린 시장(예: '2030 직장인')을 버려라.** 단 한 명의 구체적인 페르소나에 집중.

- 꾸며낸 문제(Fabricated Problem) 함정 경계
- 가장 위험한 가설(Riskiest Assumption): "이 가설이 틀리면 사업 전체가 무너지는" 문제 가설을 명확히 정의

**솔루션 우선 사고 vs 문제 우선 사고:**
- ✗ 솔루션 우선: 자신의 솔루션을 팔기 위해 고객이 겪지도 않는 문제를 인위적으로 만들어냄
- ✓ 문제 우선: 페르소나 기반으로 도출한 "고객이 진정으로 고통받고 있는 문제는 무엇인가?"

#### Step 2. 검증 방법 — Two-Track 전략

**Track 1. 데이터 기반 검증 (AI)**
- 8개 핵심 증거 소스(텍스트·영상)를 NotebookLM에 업로드
- 창업가의 개입을 철저히 차단하고 날것의 데이터 기반 객관적 페르소나 추출
- 가상 인터뷰(Simulated Personas) 진행

**Track 2. 현장 아웃리치 (Field Outreach)**
- AI로 1차 검증된 가설을 바탕으로 콜드 컨택 메시지 설계
- 실제 얼리어답터(에반젤리스트) 최소 3~5명 섭외
- 현장의 진실(Raw Data) 확인

#### The Mom Test — 좋은 인터뷰 질문 원칙

| 나쁜 질문 | 좋은 질문 |
|---------|---------|
| 미래 의향 질문: "이 제품이 나오면 얼마면 사시겠어요?" | 과거 행동 요구: "지난번에 그 문제가 발생했을 때 어떻게 대처하셨나요?" |
| 칭찬 유도: "이 아이디어 어때요?" (죽음의 키스) | 객관적 사실 탐색: "그 문제를 해결하려 얼마의 시간과 돈을 썼나요?" |
| 아이디어 피칭: 먼저 설명하고 설득 | 상황적 맥락 파악: "작업 중 가장 짜증났던 구체적인 상황은?" |

> **The 80/20 Rule:** 인터뷰어는 20% 말하고 80% 들어라. 침묵을 두려워하지 마라.

#### Step 3. 확보된 증거 — 진짜 문제를 증명하는 3가지 시그널

1. **시간과 비용 지출 (Time & Money Spent):** 고객이 이미 해당 문제를 해결하기 위해 돈이나 시간을 지불한 명확한 이력이 있는가?
2. **대안 해결책의 존재 (Alternative Solutions):** 완벽하지 않더라도 자신만의 방식(엑셀 수작업 등)으로 고통을 덜어내기 위해 행동한 흔적이 있는가?
3. **미래 참여 약속 (Commitment 확인):** 단순한 호응을 넘어, 다음 프로토타입 테스트에 시간을 내어 참여하겠다고 확언을 받았는가?

#### Step 4. 결론 — CPF 최종 판결 및 피벗/전진 결정

**[판결 1]** 가설의 객관성: AI 인터뷰와 현장 데이터를 종합했을 때, 가설이 반증될 가능성을 충분히 열어두었는가?

**[판결 2]** CPF 달성 여부: 창업가 본인의 문제가 아닌, 실제 고객이 뼈저리게 고통받는 'Customer-Problem Fit'이 입증되었는가?

```
증거가 가설을 지지하는가?
    YES → Persevere (전진): 코딩 없이 랜딩 페이지 구현 → 실제 트랙션 테스트
    NO  → Pivot (방향 전환): 페르소나 전면 수정 → 새로운 가설 재설정
```

> **Lean Startup Mindset:** 가설 기각도 Proof Day의 성공적인 성과다. 잘못된 길을 시장 출시 전에 발견하고 자본의 고갈(Burn-out)을 막아낸 것이 가장 성공적인 성과다.

## 9. 핵심 개념 및 용어 정리

| 용어 | 정의 |
|------|------|
| **Lean Startup** | 빠른 가설-실험-학습 사이클로 실패 비용을 최소화하는 창업 방법론 |
| **MVP (Minimum Viable Product)** | 핵심 가치를 검증하기 위한 최소 기능 제품 |
| **Pivot** | 핵심 가설이 반증됐을 때의 전략적 방향 전환 (Restart가 아님) |
| **Customer Discovery** | 고객과 직접 대화하며 문제/솔루션 가설을 검증하는 프로세스 |
| **CPF (Customer-Problem Fit)** | 고객이 실제로 고통받는 문제를 정확히 정의한 상태 |
| **Riskiest Assumption** | 틀릴 경우 사업 전체를 무너뜨리는 가장 치명적인 가설 |
| **Confirmation Bias** | 자신의 아이디어가 옳다는 증거만 선택적으로 수용하는 인지 편향 |
| **The Mom Test** | 엄마에게 물어봐도 거짓 긍정이 나오지 않는 인터뷰 질문 설계 원칙 |
| **Effectuation** | 미래 예측 대신 현재 자원으로 통제 가능한 결과를 만들어내는 사고방식 |
| **Fabricated Problem** | 솔루션 판매를 위해 인위적으로 만들어낸 꾸며낸 문제 |
| **Javelin Board** | 고객/문제 가설 → 실험 → 결과/학습을 추적하는 린 검증 도구 |
| **KJ법** | 인터뷰 데이터를 Bottom-up으로 클러스터링해 진짜 원인을 도출하는 친화도법 |
| **Simulated Personas** | AI에게 페르소나 데이터를 주입해 생성한 가상 인터뷰 대상자 |
| **NotebookLM** | Google의 근거형 AI — 업로드된 소스 기반으로 객관적 분석 수행 |
